= 从NetApp Data Classification中的数据源收集的元数据
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
NetApp Data Classification 在扫描数据源和系统时收集元数据。大多数元数据可用，但某些属性并非适用于所有协议。

[cols="15,25,25,25"]
|===
|  | *元数据* | *CIFS* | *NFS* 


.3+| *时间戳* | _创建时间_ | 可用 | 不可用（Linux 不支持） 


| _上次访问时间_ | 可用 | 可用 


| _上次修改时间_ | 可用 | 可用 


.2+| *权限* | _开放权限_ | 如果“EVERYONE”组有权访问该文件，则视为“对组织开放” | 如果“其他人”有权访问该文件，则视为“对组织开放” 


| _用户/组访问_ | 用户和组信息取自 LDAP | 不可用（NFS 用户通常在服务器本地进行管理，因此同一个人在每个服务器上可以有不同的 UID） 
|===
[NOTE]
====
* 数据分类不会从数据库数据源中提取“上次访问时间”。
* 较早版本的 Windows 操作系统（例如，Windows 7 和 Windows 8）默认情况下禁用"上次访问时间"属性的收集，因为它可能会影响系统性能。如果未收集此属性，基于"上次访问时间"的 Data Classification 分析将受到影响。如果需要，您可以在这些较旧的 Windows 系统上启用上次访问时间的收集。


====


== 上次访问时间

当 Data Classification 从文件共享中提取数据时，操作系统将其视为访问事件，并相应地更新"上次访问时间"。扫描后，Data Classification 会尝试将上次访问时间还原为原始时间戳。如果 Data Classification 在 CIFS 中没有写属性权限，或者在 NFS 中没有写权限，则系统无法将上次访问时间还原为原始时间戳。

默认情况下，Data Classification 在没有还原原始时间戳的写入权限时不会扫描文件。如果您的数据可以容忍上次访问时间未重置为原始时间戳，请在系统配置页面中启用 *Scan without write permissions*。

image:screenshot_scan_missing_permissions.png["显示开关的屏幕截图，即使 Data Classification 没有正确的权限将上次访问时间恢复为原始时间戳，也可使用该开关让 Data Classification 扫描您的文件。"]

此功能适用于本地ONTAP系统、 Cloud Volumes ONTAP、 Azure NetApp Files、 Amazon FSx for NetApp ONTAP管理和第三方文件共享。


NOTE: 配置了 SnapLock 的 ONTAP 卷具有只读权限，并且无法将上次访问时间还原为原始时间戳。



=== 扫描分析事件

"调查"页面中有一个名为"扫描分析事件"的筛选器，可以显示由于 Data Classification 无法还原上次访问时间而未分类的文件，也可以显示即使 Data Classification 无法还原上次访问时间也已分类的文件。

image:screenshot_scan_analysis_event_filter.png["过滤器的屏幕截图，用于查看根据是否具有写入权限扫描了哪些文件，以及未扫描哪些文件。"]

过滤器选择包括：

* "Not classified -- Cannot revert last accessed time" - 显示由于缺少写入权限而未分类的文件。
* "Classified and updated last accessed time" - 这显示了已分类的文件，且 Data Classification 无法将上次访问时间重置回原始日期。此过滤器仅适用于启用 *Scan without write permissions* 的环境。


如果需要，您可以将这些结果导出到报表中，以便查看哪些文件由于权限而正在扫描或未扫描。link:task-investigate-data.html#download-your-report["了解有关数据调查报告的更多信息"^].
